# Vitalis Care and Research Centre Platform

Hybrid MVP for a single-brand clinic platform.

## Stack

- Angular app: `apps/web`
- TypeScript Node API: `apps/api`
- Supabase: auth, PostgreSQL, realtime-ready chat tables, prescription storage, RLS
- Node API responsibility in hybrid mode: Razorpay order creation, signature verification, webhooks, and other server-only privileged logic

## Supabase Setup

Detailed steps are in `docs/supabase-setup.md`.

Razorpay payment setup steps are in `docs/razorpay-setup.md`.

1. Create a Supabase project.
2. Run `supabase/schema.sql` in the Supabase SQL editor.
3. Replace placeholder values in `apps/web/src/app/supabase.client.ts`:
   - `YOUR_PROJECT_ID`
   - `YOUR_SUPABASE_ANON_KEY`
4. Enable phone OTP and Google provider in Supabase Auth if you want those login methods live.
5. Deploy the admin-only doctor creation Edge Function:

```powershell
supabase functions deploy create-doctor
```

The function needs the standard Supabase function environment variables:

```text
SUPABASE_URL
SUPABASE_ANON_KEY
SUPABASE_SERVICE_ROLE_KEY
```

## Local Development

```powershell
npm install --prefix apps/web
npm install --prefix apps/api
npm run dev:web
npm run dev:api
```

## TypeScript API

The backend is TypeScript. Source files are in `apps/api/src`, and production output is generated by:

```powershell
npm run build:api
```

## Demo Scope

- Patient OTP register/login flow
- Staff email/password login
- Forgot/reset password flow
- Google login support through Supabase OAuth
- Patient disease selection and intake form
- Consultation history and chat UI
- Admin doctor management and manual assignment
- Doctor chat, prescription upload, and completion workflow
